1.2 Identifier Names:

The names of variables, functions, labels and various other user-defined items are called identifiers.
  • The first character must be a letter or an underscore.
  • Subsequent digits must be either letters, digits or underscores.
  • In C99, an external identifier has at least 31 significant characters and an internal identifier has at least 63 significant characters.
  • An identifier name is case-sensitive.
  • An identifier can't be a C keyword and can't have the same name as a function in the C library.

2. Formal Parameters:
If a function is to use arguments it must declare variables that will accept the values of the arguments. These variables are called the formal parameters of the function. They behave like any other local variables inside the function.


3. Global Variables:
Global variables are known throughout the program and maybe used by any piece of code. If there's a variable say - 'count' has been declared globally. Now if in a function, the variable 'count' is declared again, then the function will not make use of the global variable and instead refer to the local variable.
If a global variable and a local variable have the same name, all references to that variable in that code block in which the local variable is declared will refer to that local variable only and have no effect on the global variable.
Storage for global variables is in a fixed region of memory set aside for this purpose by the compiler. Global variables are helpful when many functions in a program use the same data. You should avoid using unnecessary global variables since they take up memory the entire time the program is executing, not just when they're needed.

1.4 The Four C Scopes:

The 2 terms - local and global are more finely subdivided in C into 4 categories:

Scope Meaning
File Scope Starts at the beginning of the file and ends with the end of the file. It refers to only those identifiers that are declared outside of all functions i.e., they're global. File scope identifiers are visible throughout the entire file.
Block Scope Begins with the opening { of a block and ends with its closing }. However, block scope also extends to function parameters in a function definition that is function parameters are included in a function's block scope variables with block scope. Variables with block scope are local to their block.
Function Prototype Scope Identifiers declared in a function prototype are visible within the prototype.
